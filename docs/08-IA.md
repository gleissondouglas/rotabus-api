# Inteligência Artificial e Modelos de Linguagem

O projeto Nuvem define seu relacionamento com IA não como uma dependência obrigatória para lógicas triviais, mas sim como uma **ferramenta acessória (Adapter)** no pipeline arquitetural, preferindo modelos determinísticos onde possível.

## 1. Funcionamento (Speech e NLP)

### Speech-to-Text (STT) e Text-to-Speech (TTS)
- Atualmente consumido diretamente via APIs do Dispositivo (Apple/Google) ou `expo-speech` para latência quase zero.
- Como evolução, o backend consumirá provedores Cloud dedicados para qualidade superior de fala (TTS Neural de voz humanizada) apenas onde o hardware local for insuficiente.

### Processamento de Linguagem Natural Local (VoiceIntentParser)
Para preservar o tempo de resposta inferior a 1 segundo e anular os custos de LLM Generativo para intenções simples (Sim, Não, Primeiro, Segundo), o sistema adota um `Parser Determinístico` no Frontend:
1. Normaliza todo áudio (lowercase, sem acento).
2. Procura palavras-chave de intenção (ex: Regex mapeando "sim", "correto", "pode ser" para a Intent `CONFIRM`).
3. Somente inputs que falham nesta detecção (ex: "Centro") são enviados ao backend como `DESTINATION_TEXT`.

## 2. A Camada "Local Intelligence"

O `LocalIntelligenceService` não é uma IA Generativa, e sim um motor de Inteligência Baseada em Regras (Rule-based).
- **Problema:** APIs do Google não entendem que "Praça do Zebu" se refere à "Praça Expozebu, Uberaba".
- **Solução:** O Local Intelligence intercepta a query transcrita e aplica _Aliases_ regionais mapeados. 
- **Benefício:** Zero custo de token, latência de microssegundos e altíssima precisão local.

## 3. Visão de Integração Futura de LLMs

Onde a inteligência determinística falhar, prevê-se o uso de LLMs Generativos (ex: OpenAI GPT-4o-mini ou Gemini Flash).

**Decisões Arquiteturais Planejadas para o LLM:**
- **Papel:** Será isolado em um Provider (`nlp.provider.js`), atuando APENAS como "Resolvedor de Intenções Complexas".
- **Contexto Passado:** Um array das últimas 3 interações e o estado atual da FSM para evitar perda de foco de prompt.
- **Cache de Semântica:** Consultas idênticas para a IA (ex: "Me leve pro centro") são cacheadas em texto para economizar tokens e dinheiro.

## 4. Prompts (Pendente de Implementação na Camada Nova)

Quando a LLM entrar na jogada, ela receberá System Prompts restritos:
*Obrigatoriamente responder em JSON. Você é o assistente de um sistema de ônibus em Uberaba. Classifique a frase do usuário em uma das intents: [CONFIRM, REPEAT, SELECT_OPTION, SEARCH_DESTINATION]. Se SEARCH_DESTINATION, extraia e limpe a localidade.*

## 5. Práticas de Custo e Tokenização
Para um projeto subsidiado ou em MVP, o controle financeiro de IA é vital:
1. Jamais enviar histórico excessivo (Manter Janela Curta no DialogManager).
2. Não usar IA para gerar TTS de resposta (o backend deve gerar `speechText` a partir de templates fixos rápidos).
3. O Backend deve atuar de _Gatekeeper_, rejeitando spam ou falas de tamanho abusivo (ex: limitando o input de texto de STT para max 150 caracteres).
